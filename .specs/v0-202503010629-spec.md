# fluster v0 Coding Spec

## Core Principles

* Portable project folder
* Single active project per server instance
* Rows → Items is 1:1
* Text-only embeddings (via `embedding_text`) and `transformers`.
* Random seed = 42 (everywhere randomness occurs)
* Jobs + job_logs with cooperative cancellation
* Plans defined in YAML
* Append-only data model
* SQLite vec0/JSON1 required (fail fast if missing)
* Assets are copied into project

# Phase 1 — Workspace + Project Layout

Workspace: `~/.fluster/`

Projects live at:

```
~/.fluster/projects/<project_name>/
  project.yaml
  plan.yaml
  project.db
  artifacts/
```

CLI:

```
fluster init <project_name>
```

Creates project folder and default `plan.yaml`.

Connection helper:

connect(project_dir) -> sqlite3.Connection

# Phase 5 — Plan YAML Schema

`plan.yaml` contains:

```
embedding:
  model_name: ...
  max_tokens: ...

reductions:
  - method: pca
    enabled: true
  - method: umap
    target_dimensions: 2
    random_state: 42
  - method: umap
    target_dimensions: 8
    random_state: 42

clustering:
  - method: hdbscan
    reduction: umap_8d
    params: { ... }

llm:
  provider: openai|ollama
  model: ...
  max_llm_calls: 200
```

CLI supports:

* manual editing
* wizard generation

# Phase 6 — ingest_rows

Command:

```
fluster ingest-rows <project_name> <csv_path>
```

Behavior:

* Create rows
* Create items (1:1)
* Copy artifacts into `artifacts/`
* Store artifact mappings

Artifacts stored content-addressed:

```
artifacts/<sha256[:2]>/<sha256>.<ext>
```

---

# Phase 7 — materialize_items

For each item:

* Extract text from artifacts
* Optional summarize
* Build `embedding_text`

`embedding_text` includes:

* row_name (if present)
* selected metadata (optional)
* summary or extracted text

Store as representation type `embedding_text`.

# Phase 11 — Exemplars (Faux-Medoids)

## cluster_exemplars

* cluster_run_id
* cluster_id
* item_id
* rank
* score

Algorithm:

* Compute centroid in original embedding space
* Pick top C nearest-to-centroid candidates
* Score by avg similarity to cluster members
* Select top K exemplars

Used for labeling.

## Final Philosophy

fluster is:

* deterministic (seed=42)
* local-first
* append-only
* auditable
* reproducible
* LLM-assisted but not LLM-driven
